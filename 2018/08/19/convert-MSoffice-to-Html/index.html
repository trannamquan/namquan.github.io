[Sharing] Converting Microsoft Office documents to HTML

Posted by Nam Quân on 19-08-2018
#msoffice #word #excel #powerpoint #c# #.net #openxml #interop #aspose #gembox
Sharing/ Programming/ C# .NET

This topic discussed about several approaches in processing, exporting Microsoft Office documents (eg. Word, Excel, Powerpoint, Visio) to HTML.
Note: Some approaches are just availabled in .Net framework.

Introduction

As far as I know, there are few solutions for processing and converting documents, and each has its own pros and cons. Hence, depending on your situation and your purpose, you can take reference this topic and make your decision.

List solutions:

  1. Interop library
  2. Open XML library
  3. Google Docs Viewer / Office Web Apps Viewer / One Drive
  4. Commercial library (Aspose, Spire, Gembox)

Solutions analysis

1. Interop library

Pros:

  • Out-of-box functions to convert directly MsWord, MsExcel, MsVisio to HTML.
  • High-quality conversion (Except headers and footers are lost).

Cons:

  • Your app running environment must have installed Microsoft Office. Thus, sometimes unexpected situations from Ms Office applications will block the whole program flow.
  • Powerpoint is not supported for converting to HTML format. Alternative is that you can convert Powerpoint to Pdf format for displaying directly on browser.
  • A very big problem for sever-side processing. If you intend to use this library in ASP.Net application, please consider this warning from Mircosoft for server-side Automation of Office.

Note: I wrote some C# code to demo several simple features of this library. Check it here

2. Open XML library

Pros:

  • Is recommended by Microsoft for replacing Interop library. Huge support documents and community.
  • Strong manipulation in document processing such as retrieving header, footer data, adding paragraphs, modifying hyperlinks inside documents…
  • No dependency on Microsoft Office.

Cons:
You need to implement HTML converting by yourself which will absolutely cost you a lots of effort and time. Luckily, Word to Html function now is supported by a library named “OpenXML Power Tools“ , but, for the moment I write this article, that function can’t parse Word chart objects to corresponding Html elements.

3. Google Doc Viewer / One Drive / Microsoft 365 Viewer

Pros:

  • Very easy to use, just call and the magic happens.
  • Many built-in features inside the documents such as searching, highlighting, printing.
  • Almost no errors, smooth running because they are implemented by Google and Mircosoft.

Cons:

  • Depend on Google and Microsoft, in case they stop providing or start charging these services.
  • Don’t know whether they can access and store our documents or not -> security problem.
  • These viewer are just for displaying office documents on browser, they can not help us in processing documents such as modifying, adding, removing data inside the documents.

Note: How to use it. Check it here.

4. Commercial library.

After have a trial with these commercial libraries: Aspose, Gembox, Spire. I would like to choose Aspose because it has many features as well as their functions work so perfect. However the Aspose Total product costs 3000$/1 developer /1 deployment. So the pros and cons below is discussed about Aspose library.

Pros:

  • Support a variety languages such as C#, C++, Java..
  • Very strong manipulation in documents processing.
  • High-quality conversion with many converting options. Support Word To Html, Excel To Html, Powerpoint To Html, Visio To Html.
  • Common problems in processing and converting documents to Html are almost solved.
  • No dependency on Microsoft Office.

Cons:

  • High-cost. You can take reference here
  • If you want specific support you need to pay them more extra fee.

Note: For detail Aspose guide, check it here.

Conclusion

Finally, you can have an overview about solutions in processing and converting MsOffice to Html. These problems are common but the solutions seem are rare and very hard to implement from scratch.

Hence, my advice is that:
If you or your team is building a large application requiring this converting to Html feature, you should buy a commercial library. It works very well, save your time and effort, so that you can focus on others important business logic in your application.

In the other hand, if your application’s documents are intend to public for everyone, and you don’t care about documents security, you can use 3rd viewer such as Google Docs Viewer or Office Web App Viewer.

The last is for people who want to learn and write your code to convert MsOffice to Html. OpenXml is a nice try, but remember what you are doing is solving a big problem which many companies spend years to handle it.

So, at this time, you know what to do next, thank you for your reading.
